# Хитрости конфигурации Rockstat

## Theia IDE

#### Длина строки pycodestyle

Добавить в корень проекта файл `config.cfg` c содержимым:

```ini
[pycodestyle]
max-line-length = 250
```

## Своё название переменных в cookies пользователей

Rockstat присваивает пользователю внутренний айдишник и при использовании Web-SDK
для трекинга событий пишет этот айдишник и в cookies на домен трекера,
и в cookies на домен сайта, и local storage. По дефолту та переменная,
которая ставится на домен трекера, называется `uid`.

В если переменная `uid` уже используется для внутренних нужд и её занимать
нельзя, а вы ставите трекер на поддомен основного сайта, то можно переименовать
переменную, которую rockstat ставит на домен трекера. Вот как:

Добавляем в Theia в папку `/project/config/custom/front` файл `config.yaml`
и пишем в него что-то типа того:

```ini
identify:
  param: rst-uid
```

В интерфейсе должно выглядеть вот так:

![custom var in cookies](/static/images/docs/examples/tip-custom-var-in-cookies.png)

Здесь `rst-uid` — это название переменной, которая будет в cookies и local storage
на домене трекера и в которой будет лежать айдишник Rockstat.
Обзываем так, чтобы она не конфликтовала с внутренними.

Помним по то, что на домене сайта переменная будет называться так,
как вы укажете в конфиге сниппета. [Подробнее про сниппет →](/docs/reference/web-sdk/snippet-and-configuration).
